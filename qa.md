# qa.md

## review 검토 규칙

- review 문서의 형식 준수와 수행일시/참조 todo 존재 여부를 최우선 확인한다.
- 완료 기준, 파일 권한 준수, 충돌 여부를 함께 검토한다.
- 남은 이슈가 있으면 done 처리하지 않는다.

## done 처리 기준

- 완료 기준 완전 충족
- todo의 선행 조건이 done 상태로 충족
- todo/role 규칙 위반 없음
- 변경 파일 목록과 리뷰 내용이 일치

## 반려 기준

- todo 미충족 (범위, 완료 기준, 검증 누락)
- 권한 규칙 위반 (수정 가능 범위 밖 파일 편집)
- 동일 파일 동시 수정 흔적
- 선행 todo 미완료 후속 반영

## 파일 수정 권한 위반 검토

- 변경 파일이 `수정 가능 파일`에 포함되는지 확인한다.
- `수정 금지 파일`이 변경되었는지 확인한다.
- 읽기 전용 파일에 대한 임의 수정 여부를 점검한다.

## 선행 조건 충족 여부 검토

- todo에서 지정한 선행 조건을 plan/todo 상태와 대조한다.
- 조건 미충족이면 즉시 todo로 되돌리거나 plan 단계 재작성 요청한다.

## plan/todo 재작성 기준

- 반복 위반 또는 불명확한 요구가 반복되면 plan 재작성
- 구현 중 경계 변경이 필요하면 plan 갱신 후 todo 재분해
- done 불가 사유가 1개 이상 발견되면 review를 완료 처리하지 않고 todo/plan로 환류
