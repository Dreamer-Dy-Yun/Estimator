# 대시보드 데모 제작 가이드

이 문서는 자사 분석/경쟁사 분석/오더 시뮬레이션 데모를 빠르게 재구성할 때, 구조와 품질을 일관되게 유지하기 위한 기준이다.  
나중에 확장(실 API 연동, 기능 추가, 스타일 고도화)할 때도 아래 원칙을 기본값으로 사용한다.

## 1) 목표 범위 정의

- 본 데모의 핵심은 **화면 흐름 검증**이다.
- 주요 흐름:
  - 상단 탭으로 페이지 이동
  - 조건 조회 UI 표시
  - 리스트/차트로 요약 확인
  - 리스트 클릭 시 상품 상세 모달 확인
- 백엔드는 실제 서버 대신 **가정 API + 하드코딩 테스트 데이터**로 동작한다.

## 2) 정보 구조(IA)와 라우팅

- 페이지 라우트는 고정:
  - `/self` : 자사 판매 분석
  - `/competitor` : 경쟁사 판매 분석
  - `/order-sim` : 오더 시뮬레이션
- 상단 네비게이션은 모든 페이지에서 동일한 위치/스타일/동작을 유지한다.
- 페이지 간 공통 구조:
  - 좌측: 조회 조건 + 보조 차트
  - 우측: 메인 리스트(테이블)

## 3) UI/스타일 시스템 기준

### 3.1 스타일 충돌 방지

- 전역 CSS는 최소화:
  - 리셋, 폰트, 디자인 토큰(색상/간격/반경)만 전역 허용
- 나머지 스타일은 전부 `CSS Modules` 사용
- 페이지별 스타일과 컴포넌트 스타일을 파일 단위로 분리

### 3.2 컬러/시각 규칙

- Primary(헤더/구분 바), Surface(컨테이너), Border, Input 배경색을 토큰으로 고정
- Self/Competitor 색상은 모든 차트에서 일관 유지
- 표 헤더/본문/강조 셀의 대비를 일정하게 유지

### 3.3 테이블 UX 규칙

- 리스트는 화면 하단까지 채우고 내부 스크롤 사용
- 헤더는 sticky 처리하여 스크롤 중에도 컬럼 맥락 유지
- 숫자는 우측 정렬, 텍스트는 좌측 정렬 원칙 적용

## 4) 공통 컴포넌트 분리 전략

중복 구현을 막기 위해 아래 컴포넌트를 먼저 분리해 둔다.

- `TopTabs`: 상단 페이지 이동 탭
- `Panel`: 섹션 박스(헤더+본문)
- `FilterCard`: 조회 조건 UI 묶음
- `DataTable`: 공통 테이블 렌더러
- `BaseModal`: 상세 모달 공통 프레임
- `ProductDetailModal`: 도메인 상세 모달

이 구조를 지키면 페이지 파일은 "데이터 연결 + 조합" 역할만 수행해 유지보수가 쉬워진다.

## 5) API 가정 및 Mock 데이터 원칙

### 5.1 API 형태

- 데이터 호출은 실제 fetch 패턴과 유사하게 `Promise` 기반으로 구현
- 권장 함수:
  - `getSelfSales()`
  - `getCompetitorSales()`
  - `getOrderSimulation()`
  - `getProductDetail(id)`

### 5.2 테스트 데이터 설계

- 최소 5개 이상의 상품 행을 제공해 리스트/차트/모달 검증 가능하게 구성
- 금액, 수량, 이익률, 재고일수 등 핵심 필드는 현실적인 범위를 유지
- 모달 상세는 최소한:
  - 상품 기본정보
  - 자사/경쟁사 비교 수치
  - 재고 추이 시계열 데이터

## 6) 페이지별 구현 체크포인트

### 6.1 자사 판매 분석

- 자사 지표 테이블 노출
- 산점도(이익률 vs 매출 등) 표시
- 행 클릭 시 상세 모달 오픈

### 6.2 경쟁사 판매 분석

- 조회 조건 아래에 비교 차트 배치
- 우측에 경쟁사 중심 리스트 배치
- 행 클릭 시 동일 상세 모달 오픈

### 6.3 오더 시뮬레이션

- 오더 추천 관련 컬럼(현재고, 추천오더량, 기대판매액 등) 표시
- 예상 실적 추이 차트 제공
- 행 클릭 시 상세 모달 연결

## 7) 데이터 표시 규칙(포맷 일관성)

- 숫자/금액/퍼센트 포맷 함수 공통화
  - `formatNumber`, `formatCurrency`, `formatPercent`
- 천단위 콤마, 단위(원, %, 개) 표기 규칙 통일
- 날짜 표기 형식도 하나로 고정

## 8) 품질 점검 루틴

- 변경 후 기본 점검:
  - 타입/번들 빌드 성공 여부
  - 린트 에러 유무
  - 주요 라우트 이동 및 모달 동작 확인
- 최소 검증 시나리오:
  1. 탭 이동 3페이지 정상
  2. 각 페이지 테이블 렌더링 정상
  3. 행 클릭 시 상세 모달 정상
  4. 스크롤 시 테이블 헤더 유지

## 9) 이후 확장 시 우선순위

1. 필터 조건과 리스트/차트 실시간 연동
2. 모달 상세 레이아웃 고도화(이미지, 입력창, AI 답변창 등)
3. 목 API를 실제 API로 교체(인터페이스 유지)
4. 반응형/접근성 개선
