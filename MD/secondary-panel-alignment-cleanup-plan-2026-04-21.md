# Secondary Panel 정합성/정리 계획

- 작성일: 2026-04-21
- 변경일: 2026-04-21
- 지시자: Yun Daeyoung

## 목표
- `product-secondary` 영역에서 기능과 맞지 않는 변수명/함수명/문구를 정합성 있게 정리한다.
- 중복 코드/불필요 코드 제거로 유지보수성을 높인다.
- CSS 중복/미사용 스타일을 줄여 스타일 구조를 단순화한다.

## 범위
- `dashboard-app/src/dashboard/components/product-secondary/**`
- 연관 타입/문구 파일(`ko.ts`, `secondaryPanelTypes.ts`) 및 필요 시 호출부

## 작업 단계
1. **정합성 점검**
   - 최근 변경된 “사이즈 통합 오더 설정/오더 수량/추천수량/확정수량” 관련 식별자 점검
   - 주석/도움말(`KO`)과 실제 계산식 일치 여부 확인
2. **명명 정리**
   - 기능과 불일치한 변수명/타입명/문구를 목적에 맞게 변경
   - 파급 영향(호출부/타입) 동시 반영
3. **중복/불필요 코드 제거**
   - 사용되지 않는 KO 키/상수/분기/보조 타입 제거
   - 불필요한 계산 중복/레거시 흔적 제거
4. **CSS 정리**
   - 현재 컴포넌트에서 사용되지 않는 클래스 제거
   - 동일/유사 역할 스타일 통합 가능한 항목 정리
5. **검증**
   - `npx tsc --noEmit`
   - 수정 파일 기준 lints 확인

## 산출물
- 정합성 반영 코드
- 불필요 코드 제거 커밋 가능한 상태(커밋은 사용자 지시 시)
