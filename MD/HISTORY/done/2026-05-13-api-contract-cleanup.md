## 목표

- 백엔드 엔드포인트가 없는 상태에서도 프론트의 API 호출부가 백엔드 계약 초안 역할을 할 수 있도록 `src/api/requests/*`에 기능 범위와 주의점을 기록한다.
- 화면/훅/컴포넌트가 mock 구현을 직접 알지 않도록 `src/api` 경계를 유지한다.
- 기능을 없애지 않는 범위에서 역할 없는 코드와 빈 UI 경계를 제거한다.
- API 계약 변경과 소스 경계 변경을 유지 문서에 같이 반영한다.

## 결과

- API 요청부는 mock 호출을 유지하지만, 실제 백엔드 연결 시 어떤 요청이 어떤 DB/세션/집계 책임을 가져야 하는지 코드 가까이 확인할 수 있게 되었다.
- 후보군 소유자 UUID는 화면으로 흘러가지 않고 request adapter에서만 세션 기준으로 붙는다.
- 상품 드로워의 판매 인사이트는 선택 경쟁 채널 필수 계약으로 좁혀져, “기본 채널로 조용히 처리”되는 여지를 줄였다.
- 빈 `PageHeader` 컴포넌트와 관련 스타일을 제거해 레이아웃 경계가 단순해졌다.
- 테스트와 빌드가 통과했다.

## 후속 조치

- 실제 백엔드가 생기면 `src/api/requests/*` 안의 mock 위임만 HTTP 요청으로 교체한다.
- 후보군 리스트 기간 기반 배지 계산은 전체 상품 분포 계산이 필요하므로, 백엔드에서 기간/채널별 집계 캐시를 검토한다.
- 후보군 업로드 템플릿 다운로드는 현재 정적 파일이지만, 운영에서는 같은 프론트 계약을 유지한 채 백엔드 다운로드 endpoint로 옮길 수 있다.
