# 후보군 추천 배지 조회와 분석 요청 상태 표시 정리

| 항목 | 내용 |
|------|------|
| 작성일 | 2026-05-18 |
| 작성 지시 | 사용자 |
| 작성자 | Codex |
| 상태 | 완료 기록 |
| 적용 범위 | `dashboard-app` 후보군 상세 모달, 자사/경쟁사 분석 요청 상태 UI, 관련 API 계약 문서 |

## 목표

- 이너 후보군에서 `조회` 이후 배지가 `추천 보기`를 눌러야만 채워지는 흐름을 없앤다.
- 추천 후보와 후보군 내부 배지는 같은 추천 API 결과를 기준으로 병합한다.
- 자사/경쟁사 분석 화면의 요청 상태가 정상 상태에서 화면 공간을 차지하지 않게 한다.

## 동작 기준

### 이너 후보군 조회

1. 후보군 상세 모달이 처음 열리면 현재 조회 데이터 기간으로 `조회`가 자동 실행된다.
2. 사용자가 조회 데이터 기간을 수정한 뒤 `조회` 버튼을 누르면 같은 조회 흐름을 다시 실행한다.
3. 기본 후보 리스트는 `getCandidateItemsByStash`로 먼저 받는다.
4. 기본 리스트가 들어온 직후 `getCandidateRecommendations`를 cursor 기반으로 끝까지 이어 받아 배지와 추천 후보를 채운다.
5. 추천 결과 중 현재 후보군에 이미 들어 있는 `skuUuid`는 이너 후보 리스트의 배지로 병합한다.
6. 추천 보기 목록에서는 현재 후보군에 이미 들어 있는 `skuUuid`를 숨긴다.
7. 총 오더 수량, 총 오더 금액, 엑셀 다운로드 DTO는 추천/배지 조회와 분리해 `subscribeCandidateOrderMetrics` SSE로 행별 갱신한다.

### 로딩/오류 표시

- 기본 리스트가 먼저 표시되고, 배지와 오더 지표는 아직 도착하지 않은 컬럼 단위로 로딩 상태를 보인다.
- 추천/배지 요청이 실패하면 배지 셀은 무한 로딩이 아니라 실패 상태로 전환한다.
- 추천 보기 버튼은 배지 계산을 시작하는 버튼이 아니라, 이미 조회된 추천 후보 목록을 여는 버튼이다.

### 분석 요청 상태

- `DashboardRequestStatus`는 자사/경쟁사 분석 API 요청 상태를 표시한다.
- 이는 `입고예정일 수집` 상태가 아니다.
- 일 단위 기본 데이터 조회가 정상 완료된 idle 상태는 화면에 표시하지 않는다.
- 갱신 중, 조회 중, 실패, 이전 데이터 표시처럼 사용자가 알아야 하는 상태만 기간 프리셋 줄의 compact 배지로 표시한다.
- 정상 상태를 숨기더라도 요청 생명주기와 stale 응답 차단은 `useDashboardRequest`가 계속 소유한다.

## 변경 결과

- 이너 후보군은 조회 직후 배지/추천 API를 자동으로 이어 호출한다.
- 추천 보기 없이도 이너 후보 리스트의 배지가 갱신된다.
- 자사/경쟁사 분석 화면의 정상 요청 상태 문구가 별도 공간을 차지하지 않는다.
- API 계약과 소스 경계 문서에 조회 순서, 추천 병합, SSE 분리, 요청 상태 표시 기준을 반영했다.

## 비목표

- 추천/배지 계산을 프론트에서 재구현하지 않는다.
- 총 오더 수량/금액을 기본 후보 리스트 API에 다시 합치지 않는다.
- 입고예정일 수집 상태와 분석 데이터 요청 상태를 같은 UI로 합치지 않는다.
