# 오더 후보군 팝업 카드 리스트 재구성 계획

- 작성일자: 2026-04-21
- 변경일자: 2026-04-21
- 지시자: Yun Daeyoung (사용자)

## 배경
- 오더 후보군 선택 팝업 내부 리스트가 단순 버튼 나열 중심이라 카드형 정보 전달이 약하고, 레이아웃/가독성이 불안정함.
- 생성 폼과 리스트가 동일 위계에서 섞여 보여 사용자 흐름(생성 -> 확인 -> 선택)이 직관적이지 않음.

## 영향 범위
- `dashboard-app/src/dashboard/components/product-secondary/ProductSecondaryPanel.tsx`
  - 후보군 선택 모달 마크업 재구성
  - 후보군 카드 항목 내 정보 계층(이름/생성일/비고/선택상태) 분리
- `dashboard-app/src/dashboard/components/product-secondary/productSecondaryPanel.module.css`
  - 모달 레이아웃, 생성 폼, 카드 리스트 스타일 전면 재정의
  - 반응형 구간에서 입력 폼 및 카드 간격/가독성 보정

## 구현 계획
1. 모달 헤더, 생성 영역, 리스트 영역을 명확히 분리한다.
2. 리스트 항목을 `button` 기반 카드로 유지하되, 내부를 타이틀/메타/설명/상태 배지로 구조화한다.
3. 선택된 후보군은 시각적으로 즉시 구분되도록 활성 스타일과 배지를 제공한다.
4. 후보군이 비어 있을 때는 빈 상태 안내 카드를 표시한다.
5. CSS에서 고정 폭/높이 의존을 줄이고 스크롤 영역과 반응형 레이아웃을 안정화한다.

## 검증 계획
- 후보군 선택 버튼 클릭 시 모달 오픈/닫힘 정상 동작 확인
- 후보군 생성 후 리스트 갱신 및 생성된 항목 선택 가능 여부 확인
- 선택 항목의 활성 상태 표현(배지/테두리/배경) 확인
- 비고가 없거나 길 때도 카드 레이아웃이 깨지지 않는지 확인
