# 자사 판매 분석 페이지 구현 요구사항

이 문서는 스크린샷 기준의 **자사 판매 분석** 단일 페이지를 구현하기 위해 필요한 항목을 정리한다.  
목적은 "바로 개발 가능한 수준의 체크리스트/명세"를 제공하는 것이다.

## 1) 페이지 목적

- 자사 상품의 판매/이익 성과를 테이블과 산점도로 동시에 보여준다.
- 필터(조회 조건)로 대상 범위를 좁히고, 우측 리스트에서 상품별 상세 성과를 확인한다.
- 리스트 행 클릭 시 상품 상세 분석 모달로 확장 탐색할 수 있어야 한다.

## 2) 레이아웃 구조

### 2.1 상단 공통 영역

- 상단 탭 버튼:
  - `자사 분석`
  - `경쟁사 분석`
  - `오더 후보군`
- 현재 페이지는 `자사 분석` 활성 상태.
- 우측에 `선택 닫기` 버튼 배치.

### 2.2 메인 본문

- 좌우 2컬럼 구조:
  - 좌측(약 25~30%): 조회 조건 + 산점도
  - 우측(약 70~75%): 판매/이익 테이블
- 우측 리스트는 페이지 하단까지 확장되어야 하며, 행이 많으면 내부 스크롤.
- 테이블 헤더는 sticky 처리(스크롤 중 상단 고정).

## 3) 좌측 조회 조건(필터) 구성

필수 표시 항목:

- 조회기간: 시작일 ~ 종료일
- 브랜드
- 카테고리
- 세부 카테고리
- 판매량 범위(최소~최대)
- 총 판매액 범위(최소~최대)
- 평균 영업이익율 범위(최소~최대)
- 총 영업이익 범위(최소~최대)

UI 규칙:

- 라벨 셀은 진한 배경 + 흰색 텍스트
- 입력 셀은 옅은 노랑 배경
- 단위(`개`, `원`, `%`)는 우측 보조 셀로 분리

## 4) 우측 메인 테이블 구성

### 4.1 컬럼 그룹

- `구분`
  - 선택
  - 순위
  - 백분위
  - 브랜드
  - 카테고리
  - 품번
  - 상품명
- `판매`
  - 평균 판매가
  - 판매량
  - 총 판매액
  - 평균원가
- `이익`
  - 평균 매출이익율
  - 총 매출이익액
  - 평균 수수료율
  - 평균 영업이익율
  - 총 영업이익액

### 4.2 테이블 동작

- 기본 정렬: 순위 오름차순
- 숫자 컬럼: 우측 정렬
- 텍스트 컬럼: 좌측 정렬
- 행 클릭: 상품 상세 모달 오픈
- 선택 컬럼(`TRUE/FALSE`)은 체크박스로 대체 가능

## 5) 산점도(좌측 하단 차트) 요구사항

- X축: 영업이익율(%) 또는 백분위 기준 선택 가능
- Y축: 총 판매액(백만 단위)
- 기준선 2개 이상 표시:
  - 기준 이익율(예: 0%)
  - 목표 이익율(예: 6~7%)
- 점 데이터는 테이블 행과 동일 상품 기준으로 매핑
- 점 hover 시 툴팁:
  - 상품명
  - 판매액
  - 영업이익율

## 6) 데이터 모델(프론트 기준)

```ts
type SelfSalesRow = {
  id: string
  selected: boolean
  rank: number
  percentile: number
  brand: string
  category: string
  type: string
  name: string
  avgPrice: number
  qty: number
  amount: number
  avgCost: number
  grossMarginRate: number
  grossMarginAmount: number
  feeRate: number
  opMarginRate: number
  opMarginAmount: number
}
```

## 7) API 가정(목 데이터)

### 7.1 조회 API

- `GET /api/self-sales`
- query 예시:
  - `from`, `to`
  - `brand`
  - `category`
  - `subCategory`
  - `qtyMin`, `qtyMax`
  - `amountMin`, `amountMax`
  - `opRateMin`, `opRateMax`
  - `opAmountMin`, `opAmountMax`

응답:

```json
{
  "items": [],
  "summary": {
    "count": 0
  }
}
```

### 7.2 상세 API

- `GET /api/product/:id/detail`
- 행 클릭 시 호출
- 상세 모달에서 사용

## 8) 숫자/포맷 규칙

- 금액: 천단위 콤마 + 원(`1,309,000,000`)
- 수량: 천단위 콤마
- 비율: 소수점 1자리 + `%`
- 차트 축 단위는 표기 축약(백만/천) 허용하되 툴팁은 원본 단위 유지

## 9) 스타일/컴포넌트 분리 가이드

- 전역 CSS 최소화(리셋 + 토큰)
- 컴포넌트 스타일은 CSS Modules 사용
- 우선 분리할 공통 컴포넌트:
  - `TopTabs`
  - `FilterCard`
  - `DataTable`
  - `Panel`
  - `BaseModal`

## 10) 구현 순서(권장)

1. 고정 레이아웃(상단/좌측/우측) 구성
2. 조회 조건 카드 마크업 및 스타일
3. 테이블 컬럼 및 더미 데이터 바인딩
4. 산점도 연결(테이블 데이터 재사용)
5. 행 클릭 상세 모달 연결
6. 필터값과 데이터 재조회 연동
7. 리스트 하단 채움 + sticky header + 내부 스크롤 검증

## 11) 완료 기준(Definition of Done)

- 상단 탭/타이틀/조회조건/테이블/산점도가 스크린샷 흐름과 동일
- 테이블이 화면 하단까지 채워지고 내부 스크롤 정상 동작
- 행 클릭 시 상세 모달이 정상 표시
- 목 API 호출 흐름이 살아 있고 하드코딩 데이터가 연결됨
- 빌드/린트 에러 없음
