# Snapshot 후보군 팝업 리스트 재구성 계획

- 작성일자: 2026-04-21
- 변경일자: 2026-04-21
- 지시자: Yun Daeyoung (사용자)

## 문제 인식
- `SnapshotConfirmPage`의 후보군 상세 팝업이 인라인 스타일 위주로 구성되어 카드 리스트가 불안정하게 보임.
- 로딩 상태와 빈 상태가 명확히 분리되지 않아 데이터가 늦게 올 때 "없음"처럼 보이는 UX 문제 존재.

## 영향 범위
- `dashboard-app/src/dashboard/pages/SnapshotConfirmPage.tsx`
  - 후보군 상세 팝업 렌더 구조 재정의
  - 로딩/빈상태/검색 결과 없음 상태 분리
  - 후보군 아이템 카드 정보 계층 정리
- `dashboard-app/src/dashboard/pages/SnapshotConfirmPage.module.css` (신규)
  - 팝업 오버레이, 다이얼로그, 검색 영역, 카드 리스트 스타일 정의

## 구현 계획
1. 팝업 전용 CSS 모듈을 추가하고 인라인 스타일을 제거한다.
2. `detailLoading` 상태를 도입해 API 호출 중 표시를 명확히 한다.
3. 후보군 상품 목록을 카드형 리스트로 재구성하고 메타 정보를 분리한다.
4. 검색/빈상태/로딩 상태를 각각 별도 UI로 처리한다.

## 검증 계획
- 후보군 클릭 시 팝업 열림 + 로딩 문구 노출 확인
- 데이터 수신 후 카드 리스트 렌더 확인
- 검색어 입력 시 필터링 결과 반영 확인
- 결과 없음/데이터 없음 시 안내 UI 정상 표시 확인
