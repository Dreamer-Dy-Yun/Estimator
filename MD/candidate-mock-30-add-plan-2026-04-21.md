# 후보군 목업 30개 추가 계획

- 작성일자: 2026-04-21
- 변경일자: 2026-04-21
- 지시자: Yun Daeyoung (사용자)

## 목적
- 오더 후보군 목록 스크롤 동작 검증을 위해 목업 후보군 데이터를 대량(30개) 추가한다.

## 변경 범위
- `dashboard-app/src/api/mock.ts`
  - 초기 후보군 seed 배열에 자동 생성 목업 30개를 합쳐 반환
  - 각 후보군에 대응하는 아이템 seed도 함께 생성

## 구현 방식
1. 기존 고정 4개 seed는 유지한다.
2. `Array.from` 기반으로 후보군 30개를 생성해 기존 배열에 합친다.
3. 동일하게 후보군 아이템 30개도 생성한다.
4. 생성일/변경일은 순차 시간으로 넣어 정렬/스크롤 테스트가 자연스럽게 보이도록 한다.

## 확인 항목
- 후보군 목록에서 스크롤 길이가 충분히 생기는지
- 이름/비고 검색 및 생성일/변경일 정렬이 대량 데이터에서도 정상인지
