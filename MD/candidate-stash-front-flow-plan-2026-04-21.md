# 후보군 스태시 프론트 플로우 계획

- 작성일: 2026-04-21
- 변경일: 2026-04-21
- 지시자: 윤대영

## 목적

`오더 담기(내용담기)` 클릭 시, 백엔드 없이 프론트 목업 상태에서 다음 플로우를 구현한다.

1. 오더 리스트(CANDIDATE_STASH)를 조회한다.
2. 리스트가 비어 있으면 `후보군 생성` 버튼을 노출한다.
3. `후보군 생성` 클릭 시 후보군(CANDIDATE_STASH)과 후보 아이템(CANDIDATE_ITEM)을 생성한다.

## 구현 범위

- API 타입 확장
  - 후보군 요약 타입
  - 후보군 생성 요청/응답 타입
- mock API 확장(localStorage 기반)
  - 후보군 목록 조회
  - 후보군 + 후보 아이템 생성
- API client/index export 확장
- 2차 드로워 UI 로직
  - `오더 담기` 클릭 시 목록 조회
  - 목록 없음 상태에서만 `후보군 생성` 버튼 노출
  - 생성 결과 메시지 표시

## 비범위

- 실제 PostgreSQL/백엔드 연동
- 서버 마이그레이션/DDL 적용

## 참고

- 상세 DB 스키마는 사용자 제공 명세를 준수하되, 현재 단계는 프론트 목업(localStorage) 저장 구조로 대응한다.
