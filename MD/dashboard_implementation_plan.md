# 대시보드 구현 실행 계획

이 문서는 아래 요구사항 문서를 실제 코드 작업으로 옮기기 위한 **통합 실행 계획**이다.

- `self_sales_page_requirements.md`
- `competitor_sales_page_requirements.md`
- `product_detail_analysis_requirements.md`
- `demo_dashboard_guidelines.md`

---

## 1) 구현 목표

- 메인 페이지(자사/경쟁사/오더 후보군 등)와 상품 상세 모달을 하나의 일관된 프론트 앱으로 구현한다.
- API는 가정된 스펙을 유지하되, 초기 단계는 목(mock) 데이터로 동작시킨다.
- CSS 충돌 없이 확장 가능한 구조(CSS Modules + 공통 컴포넌트)로 구축한다.

---

## 2) 개발 범위(이번 사이클)

### 포함 범위

- 공통 레이아웃/스타일 토큰/네비게이션
- 조회 조건 카드/테이블/차트/모달 공통 컴포넌트
- 메인 페이지 UI + 기본 동작
- 상품 상세 분석 모달(핵심 블록)
- mock API 계층 + 테스트 데이터
- 빌드/린트 기준 통과

### 제외 범위(후속)

- 백엔드 실 API 연동
- 권한/로그인
- 고급 상태관리(전역 캐시 최적화)
- 실시간 협업/알림

---

## 4) 아키텍처 설계 원칙

## 4.1 프론트 구조

- `src/pages`: 라우트 페이지
- `src/components/common`: 공통 UI 컴포넌트
- `src/components/domain`: 상세 도메인 컴포넌트(필요 시)
- `src/api`: API 인터페이스 + mock 구현
- `src/styles`: 토큰/전역 최소 CSS
- `src/utils`: 포맷/계산 유틸

## 4.2 스타일 원칙

- 전역 CSS는 reset + tokens만 유지
- 개별 스타일은 CSS Modules로 한정
- 공통 색상/간격/타이포 토큰 사용 강제

## 4.3 데이터 원칙

- 페이지는 API 레이어만 호출
- 컴포넌트는 전달받은 데이터 렌더링에 집중
- 숫자 포맷/계산은 util 함수로 분리

---

## 5) 단계별 실행 계획

## Phase 0. 착수 준비

- 목표: 작업 기반 정리
- 작업:
  - 프로젝트 스캐폴드 확인
  - 라우트 기본 틀 구성
  - lint/build 기본 실행 확인
- 완료 기준:
  - `npm run build` 성공
  - 기본 라우트 페이지 이동 가능

## Phase 1. 디자인 시스템/공통 컴포넌트

- 목표: CSS 충돌 없는 기반 구축
- 작업:
  - tokens/base 스타일 작성
  - `TopTabs`, `Panel`, `FilterCard`, `DataTable`, `BaseModal` 구현
  - sticky header, 내부 스크롤, 화면 하단 채움 규칙 적용
- 완료 기준:
  - 공통 컴포넌트로 1개 샘플 페이지 조립 가능
  - 스타일 충돌 없음

## Phase 2. API 목 계층 + 더미 데이터

- 목표: 페이지 구현 가능 데이터 준비
- 작업:
  - self/competitor/order/detail mock 함수 구현
  - 타입 정의(`types.ts`) 정리
  - 응답 지연(sleep) 포함해 비동기 호출 형태 유지
- 완료 기준:
  - 각 API 함수 단독 호출 시 정상 데이터 반환
  - 타입 에러 없음

## Phase 3. 자사 판매 분석 페이지 구현

- 목표: 자사 분석 화면 완성
- 작업:
  - 좌측 조회 조건 + 산점도
  - 우측 판매/이익 테이블
  - 행 클릭 시 상세 모달 연결
- 완료 기준:
  - 요구사항 문서 기준 UI/동작 충족
  - 리스트 하단 채움/내부 스크롤 정상

## Phase 4. 경쟁사 판매 분석 페이지 구현

- 목표: 경쟁사 비교 화면 완성
- 작업:
  - 좌측 조회 조건 + 가로 막대 비교 차트
  - 우측 경쟁사/자사 비교 테이블
  - 행 클릭 상세 모달 공용 재사용
- 완료 기준:
  - 좌측 차트가 조회 조건 아래 배치
  - 우측 리스트 스크롤/헤더 고정 정상

## Phase 5. 상품 상세 분석 모달 구현

- 목표: 상세 의사결정 화면 완성
- 작업:
  - 상단 이미지/기본정보/성과요약
  - KPI 블록(예측/재고/오더/기대실적)
  - 기간별 판매 차트, 재고 추이, 사이즈 표
  - 오더량 변경 입력 시 재계산
- 완료 기준:
  - 리스트에서 모달 오픈/닫기 정상
  - 핵심 차트/표 렌더링 정상

## Phase 6. 통합 점검/마감

- 목표: 데모 품질 보장
- 작업:
  - 라우트 전환 시나리오 점검
  - 포맷/정렬/단위 표기 통일 점검
  - 빌드/린트 최종 확인
- 완료 기준:
  - 오류 없이 실행
  - 문서 요구사항 대비 체크리스트 충족
