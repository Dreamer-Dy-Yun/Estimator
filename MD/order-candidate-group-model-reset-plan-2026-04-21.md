# 오더 후보군 모델 리셋 계획

- 작성일: 2026-04-21
- 변경일: 2026-04-21
- 지시자: 윤대영

## 변경 배경

기존: 하나의 상품 단위로 후보군을 보유  
변경: 하나의 후보군(상품군)을 만들고, 그 안에 각 상품을 1개씩 등록

## 설계 요점

1. 후보군 조회는 상품 ID 필터 없이 전체 후보군 조회
2. 후보군 아이템은 `stashUuid + productId` 기준으로 유니크(중복 등록 방지)
3. 오더 후보군 페이지는 후보군 카드 리스트 중심으로 재구성
4. 카드 상세에는 후보군에 담긴 상품 목록(코드/상품명/등록시각) 표시

## 구현 범위

- API 타입/계약 업데이트
- mock 저장/조회/삭제 로직 업데이트
- 2차 드로워 후보군 선택 로직 업데이트(전체 후보군 사용)
- 오더 후보군 페이지(SnapshotConfirmPage) 리스트를 후보군 중심 카드 구조로 재작성
