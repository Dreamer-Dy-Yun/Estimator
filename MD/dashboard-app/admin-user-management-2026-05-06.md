# 관리자 유저 관리 화면 도입

| 항목 | 내용 |
|------|------|
| 작성일 | 2026-05-06 |
| 상태 | 반영 완료 |

## Goal

대시보드 업무 탭과 분리된 관리자 화면을 만들고, 관리자 권한에서 유저 정보를 관리할 수 있게 한다.

## Scope

- `/admin` 별도 라우트 추가
- 관리자 권한 보호 라우트 추가
- 헤더 우상단 관리자 진입 버튼 추가
- mock 유저 목록 조회, 추가, 제거와 로그인 ID/권한/활성 상태 수정
- API 계약과 문서 갱신

## Principles

- 관리자 화면은 대시보드 탭 안에 넣지 않는다.
- 화면은 mock 구현을 직접 import하지 않고 `src/api` 계약만 호출한다.
- 현재 로그인한 관리자 본인의 권한 해제와 비활성화는 목 단계에서도 막아 잠금 상태를 만들지 않는다.
- 실제 백엔드 전환 시 `AuthApi`의 관리자 유저 관리 메서드 구현만 교체할 수 있게 둔다.

## Plan

1. `src/api/types/auth.ts`에 관리자 유저 목록/추가/제거/수정 타입을 추가한다.
2. `api/mock/authApi.ts`에서 mock 유저 목록을 `localStorage`에 저장한다.
3. `RequireAdmin`으로 `/admin` 접근을 관리자 세션에만 허용한다.
4. `src/admin/AdminUsersPage.tsx`에서 유저 목록을 추가/제거/편집 가능하게 표시한다.
5. 대시보드 헤더에는 관리자 권한일 때만 `/admin` 진입 버튼을 표시한다.
6. 테스트와 빌드 후 배포한다.

## Result

관리자 계정은 대시보드 헤더에서 관리자 화면으로 이동할 수 있고, `/admin`에서 로그인 ID와 초기 비밀번호로 목 유저를 추가하고, UUID 기준으로 제거/수정한다. 사용자 엔티티의 유일한 식별자는 서버 생성 UUID이고, 로그인 ID는 인증용 계정 값이다. 이메일 초대 발송과 표시 이름 관리는 현재 범위에 없다. 변경 결과는 브라우저 `localStorage`에 유지된다.

## Non-goals

- 비밀번호 초기화
- 조직/팀 단위 권한 모델
- 감사 로그
