# 후보군 담기 및 스냅샷 흐름 변경 원문

- 작성 방법 지시: 사용자
- 작성자: Codex
- 작성일: 2026-05-13

## Goal

자사/경쟁사 분석에서 바로 2차 드로워로 후보군 스냅샷을 만들던 흐름을 끊고, 후보군 담기와 스냅샷 확정을 이너후보군 흐름으로 분리한다.

## Original Request

현재 자사/ 경쟁사에서 2차 드로워 열어서 후보군 위치를 선택하고 후보군에 스냅샷을 추가 하도록 되어 있는데, 이것을 변경하려 한다.

먼저,

자사/경쟁사 탭

- 더이상 2차 드로워를 열지 않는다. 이너 후보군에서는 여전히 열림.
  - 드로워 컴포넌트에서 2차 드로워 펼침 매개변수로 받아 처리한다.
  - 따라서 여는 버튼(반원)도 생성하지 않는다. esc키를 통한 닫기는 지원한다.
  - 즉 코드에서 삭제하는 것이 아니라 추후 사용할 가능성을 보아 살려는 둔다.
- 후보군 선택/저장
  - 리스트에서 체크박스를 둔다.
  - 헤더에는 전체 선택 체크박스가 존재한다.
  - 선택한 물품을 후보군으로 라는 버튼을 만들고 여기서 후보군을 선택하거나 생성하여 담을 수 있도록 한다.

이너후보군

- 조회 범위
  - 이제 이너후보군의 리스트상의 값은 조회 범위에 따라 실시간으로 계산된다.
  - 이로 인해 백엔드 처리량이 늘어 날 수 있다. 이 부분에 대해서는 API 호출부에 명시하고 제안하라.
- 스냅샷
  - 스냅샷은 이제 이너후보군내의 2차 드로워에서만 작성되게 된다.
  - 원래 리스트에서 초기 작성하려 하였으나, 날짜 설정에 따라 값이 달라지므로, 이는 하지 않기로 한다.
  - 이너후보군 내의 2차 드로워에서만 개별 작성할 수 있고, 작성하게 된다.
  - 스냅샷은 기존과 같은 저장 범위(그래프 외 전 영역)를 보관하지만, 화면상 표기는 AI 코멘트와 각 사이즈별 오더량만 표기한다.
  - 스냅샷의 저장 범위가 넓은 이유는, 추후 저장 당시의 판단의 근거를 남겨 두기 위함이다.
  - 2차 드로워의 헤더 부분에 스냅샷 정보 보기(더 나은 어휘 있으면 변경) 체크박스를 둔다.
  - 스냅샷이 없으면 비활성화 하라.
  - 스냅샷 저장 버튼은 기존 스냅샷이 없으면 "저장", 있으면 "수정"으로 한다.
- 리스트 컬럼에 상세확정 여부를 표시하는 컬럼을 둔다.
  - 상세 확정은 각 발주량을 조정하고 확정한 경우, 즉 스냅샷이 존재하는 경우이다.

드로워

- 드로워 조작을 변경한다.
  - 이제 좌/우 방향키는 각각 드로워 열기, 닫기가 된다.
  - 리스트 엘리먼트 상에서 좌 키 한번이면 1차, 거기서 한번 더 누르면 2차 드로워가 열린다.
  - 닫힘은 그 반대.
  - 상/하 방향키는 이전/이후 아이템으로 이동한다.

추가 요청

- 자사 분석과 경쟁사 분석에서 색상 필터도 넣는다.
- 순서는 지금까지의 색상 순서와 같이 한다.

## Non-goals

- 기존 2차 드로워 코드를 삭제하지 않는다.
- 이너후보군의 2차 드로워 동작은 유지한다.
