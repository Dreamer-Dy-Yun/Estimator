# 후보군 담기 및 스냅샷 흐름 변경 계획

- 작성 방법 지시: 사용자
- 작성자: Codex
- 작성일: 2026-05-13

## Goal

정리 문서를 기준으로 자사/경쟁사 분석, 이너후보군, 드로워, API 계약을 한 흐름으로 맞춘다.

## Plan

- [x] 현재 드로워/후보군/API 호출 경계를 확인한다.
- [x] 자사/경쟁사 공통 필터에 `colorCode` 필터를 추가한다.
- [x] 자사/경쟁사 분석 리스트에 체크박스와 전체 선택 체크박스를 추가한다.
- [x] `선택한 물품을 후보군으로` 버튼과 후보군 선택/생성 모달을 추가한다.
- [x] 후보군 담기 API를 스냅샷 저장과 분리하고, API 계약/목업을 정리한다.
- [x] ProductDrawer에 2차 드로워 허용 여부 매개변수를 추가한다.
- [x] 자사/경쟁사 탭에서는 2차 드로워 버튼과 2차 드로워 진입을 비활성화한다.
- [x] 이너후보군에서는 기존처럼 2차 드로워를 열 수 있게 유지한다.
- [x] 이너후보군 리스트 조회 API에 데이터 참조 기간을 전달하고, API 호출부에 백엔드 부하 주의 사항을 남긴다.
- [x] 이너후보군 리스트에 상세확정 여부 컬럼을 추가한다.
- [x] 2차 드로워에서 스냅샷 정보 보기 토글과 저장/수정 버튼명을 정리한다.
- [x] 라이브 계산 화면은 AI 코멘트와 사이즈별 오더량 중심으로 유지하고, 스냅샷 보기 토글 상태에서는 저장 당시 전체 값과 기간을 표시한다.
- [x] 방향키 조작을 `좌=열기`, `우=닫기`, `상/하=이전/다음`으로 변경한다.
- [x] 관련 문서와 API 스펙을 업데이트한다.
- [x] `npm run test:run`, `npm run build`, 브라우저 확인을 수행한다.

## Non-goals

- 2차 드로워 코드를 삭제하지 않는다.
- 백엔드 실제 엔드포인트를 구현하지 않는다.
- 스냅샷 저장 범위를 축소하지 않는다.
