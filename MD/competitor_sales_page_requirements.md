# 경쟁사 판매 분석 페이지 구현 요구사항

이 문서는 스크린샷 기준의 **경쟁사 판매 분석** 단일 페이지 구현 항목을 정리한다.  
목적은 화면 구조, 데이터 모델, API 가정, 완료 기준을 명확히 정의해 빠르게 개발 가능한 상태를 만드는 것이다.

## 1) 페이지 목적

- 경쟁사(예: 크림)와 자사 성과를 동일 상품 기준으로 비교한다.
- 좌측 필터/차트로 빠르게 경향을 파악하고, 우측 테이블에서 상세 수치를 확인한다.
- 상품 행 클릭 시 상세 분석 모달을 열어 추가 판단이 가능해야 한다.

## 2) 레이아웃 구조

### 2.1 상단 공통 영역

- 상단 탭 버튼 3개:
  - `자사 분석`
  - `경쟁사 분석` (현재 활성)
  - `오더 시뮬레이션`
- 페이지 상단 우측 `선택 닫기` 버튼 배치
- 메인 타이틀: `경쟁사 판매 분석`

### 2.2 본문 2컬럼

- 좌측(약 25~30%):
  - 조회 조건 카드
  - 비교 막대 차트(조회 조건 하단)
- 우측(약 70~75%):
  - 경쟁사/자사 비교 테이블

동작 규칙:

- 우측 리스트는 화면 하단까지 채움
- 행이 많으면 내부 스크롤
- 테이블 헤더 sticky 유지

## 3) 좌측 조회 조건 구성

필수 필드:

- 조회기간(시작일 ~ 종료일)
- 브랜드
- 카테고리
- 세부 카테고리
- 판매량 범위(최소~최대)
- 총 판매액 범위(최소~최대)

UI 규칙:

- 라벨 영역: 진한 배경 + 흰 글자
- 입력 영역: 옅은 배경(노랑 톤)
- 단위(`개`, `원`) 분리 표기

## 4) 좌측 비교 차트 요구사항

- 형태: 가로 막대 차트(상품 품번별 비교)
- 시리즈:
  - `크림`(경쟁사)
  - `자사`
- 기준:
  - 동일 품번 단위로 값 비교
  - 축 단위는 백만 단위 축약 가능
- 툴팁:
  - 품번/상품명
  - 경쟁사 값
  - 자사 값

## 5) 우측 메인 테이블 구성

### 5.1 컬럼 그룹

- `구분`
  - 선택
  - 순위
  - 백분위
  - 브랜드
  - 카테고리
  - 품번
  - 상품명
- `크림`
  - 평균 판매가
  - 판매량
  - 총 판매액
- `자사`
  - 평균 판매가
  - 판매량
  - 총 판매액

### 5.2 동작

- 기본 정렬: 순위 오름차순
- 숫자 우측 정렬, 텍스트 좌측 정렬
- 선택 컬럼은 체크박스 또는 TRUE/FALSE 표기 허용
- 행 클릭 시 상품 상세 모달 오픈

## 6) 데이터 모델(프론트 기준)

```ts
type CompetitorCompareRow = {
  id: string
  selected: boolean
  rank: number
  percentile: number
  brand: string
  category: string
  type: string
  name: string
  competitorAvgPrice: number
  competitorQty: number
  competitorAmount: number
  selfAvgPrice: number | null
  selfQty: number | null
  selfAmount: number | null
}
```

## 7) API 가정(목 데이터)

### 7.1 리스트 조회 API

- `GET /api/competitor-sales`
- query 예시:
  - `from`, `to`
  - `brand`
  - `category`
  - `subCategory`
  - `qtyMin`, `qtyMax`
  - `amountMin`, `amountMax`

응답 예시:

```json
{
  "items": [],
  "summary": {
    "count": 0
  }
}
```

### 7.2 상세 API

- `GET /api/product/:id/detail`
- 테이블 행 클릭 시 호출
- 상세 모달 공용 데이터로 사용

## 8) 포맷 규칙

- 금액: 천단위 콤마 (예: `1,309,000,000`)
- 수량: 천단위 콤마
- 백분위: 소수점 1자리
- null 값은 `-`로 표시

## 9) 스타일/컴포넌트 분리 기준

- 전역 CSS는 리셋/토큰만 허용
- 페이지 및 컴포넌트 스타일은 CSS Modules 사용
- 재사용 컴포넌트:
  - `TopTabs`
  - `FilterCard`
  - `Panel`
  - `DataTable`
  - `BaseModal` / `ProductDetailModal`

## 10) 구현 순서(권장)

1. 상단 탭/타이틀/2컬럼 레이아웃 구성
2. 좌측 조회 조건 카드 구성
3. 우측 비교 테이블 컬럼/데이터 바인딩
4. 좌측 가로 막대 차트 연결(테이블 데이터 재활용)
5. 행 클릭 모달 연결
6. 테이블 내부 스크롤 + sticky 헤더 적용
7. 필터-조회 연동

## 11) 완료 기준(Definition of Done)

- 스크린샷과 동일한 정보 구조(좌측 조건+차트, 우측 테이블) 완성
- 경쟁사/자사 값이 동일 품번 기준으로 비교 표시
- 테이블이 화면 하단까지 채워지고 스크롤 동작 정상
- 행 클릭 시 상세 모달 정상 표시
- 목 API 호출 흐름 유지
- 빌드/린트 오류 없음
