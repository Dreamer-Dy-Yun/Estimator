# sub-agent.md

## todo 참조 규칙

- 할당된 todo 문서만 작업 근거로 사용한다.
- todo의 작업 범위 외 파일은 수정하지 않는다.
- 입력/출력 계약 항목이 미비한 경우 Sub-Agent는 변경 없이 QA 질문 항목에 남긴다.

## 작업 수행 규칙

- 수정 가능 파일만 편집하고, 생성 가능 파일만 추가한다.
- 수행 단계는 todo의 체크리스트 순서를 따른다.
- 작업 중 추가 필요 파일이 생기면 바로 수정하지 않고 review에 이슈로 남긴다.

## 파일 수정 제한

- 읽기 전용 파일은 참조용으로만 사용한다.
- 수정 금지 파일은 절대 건드리지 않는다.
- 동시 수정 충돌을 피하기 위해 todo를 넘나들며 파일을 추가하지 않는다.

## 생성 가능 파일 제한

- review 문서만은 `mulAg/md/review/` 경로에 생성 가능하다.
- 기타 산출물 파일은 Orchestrator 지시가 없으면 생성하지 않는다.

## 작업 범위 이탈 시 처리 방식

- 범위 밖 이슈 발견 시 작업 중지 없이:
  - 어떤 todo에서 발생했는지 정리
  - 필요한 파일/범위와 이유 기록
  - 바로 변경 없이 `남은 이슈` 또는 `QA 확인 요청 사항`에 등록

## review 작성 규칙

- review는 `mulAg/md/review/REVIEW-###.md` 형식으로 작성한다.
- 수행 일시는 `YYYY-MM-DD HH:mm:ss` 형식으로 기재한다.
- 아래 항목을 반드시 포함한다.
  - 수행 일시
  - 참조한 todo
  - 수행 내용
  - 변경 파일
  - 생성 파일
  - 미변경 파일
  - 검증 내용
  - 남은 이슈
  - QA 확인 요청 사항
